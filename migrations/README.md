# Database migrations

Alembic is the source of truth for schema changes. The SQLAlchemy models in
`app/database/models.py` are supplied to Alembic for migration autogeneration.

## First use on the existing database

Install project dependencies in the activated virtual environment, then run:

```powershell
python -m pip install -e .
alembic upgrade head
```

The initial revision is safe for the existing InstaOS database: if `files`
already exists, Alembic records the baseline revision and does not alter data
or the table. A fresh database receives the exact current `files` schema.
This baseline revision deliberately cannot be downgraded, because it cannot
distinguish a table it created from a pre-existing production table. Restore a
backup instead of attempting to downgrade the baseline.

## Future schema changes

1. Change the SQLAlchemy model deliberately.
2. Generate a candidate migration:

   ```powershell
   alembic revision --autogenerate -m "describe the change"
   ```

3. Review the generated file in `migrations/versions/`; never apply an
   autogenerated migration without review.
4. Apply it locally:

   ```powershell
   alembic upgrade head
   ```

5. Commit the model change and its migration in the same change set.

Use `alembic current` to see the applied revision and `alembic history` to
inspect the migration chain. Do not edit an already-applied migration; add a
new revision instead.
